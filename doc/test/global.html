cbuf_push_contract(orig, x, next) → {Boolean}

Specification (contract) of push operation Essentially, the specification says that
  • either orig is full and then contents(next) == contents(orig)
  • or contents(next) == array_push(contents(orig), x)
Of course, the invariant must be true before and after the operation.

cbuf_shift_contract(orig, x, next) → {Boolean}

Specification (contract) of the 'shift' operation. Essentially, the specification says that:
  • either orig is empty: x === undefined && next === orig
  • or contents(orig) == x ++ contents(next) where '++' means ``append''
Naturally, both orig and next should satisfy the invariant.

contents(cbuf) → {Array}

The contents of a circular buffer is an array of elements, starting with the oldest (FIFO, CBUF.shift will return the starting element) and ending with the most recently added. This corresponds to the FIFO operations on the buffer: CBUF.push will append at the end while CBUF.shift will remove from the front. This function will return this contents as an array of elements in FIFO order.

interval(from, size, n) → {Array}

Interval(from, size, n): generates the indices in the DATA array of a circular buffer that
  • has 'from' as the index of the first element
  • and that contains 'size' elements out of
  • and has an underlying array of size 'n'
E.g. for n = 3, from = 2, size = 2, we'll obtain [ 2, 0 ]. consisting of 'buf[2] buf[0]' represented by the interval [2..1[. The result is obtained as follows:
[2 .. (2 + size)[ = [2 .. 4 [
=> mod (n == 3) => [2 .. 1 [
The precondition is
from >= 0 && n > 0 && 0 >= size < n.
Parameters:
Name Type Description
from Integer starting index
size Integer number of elements in the buffer
n Integer maximal size of the (array underlying the) buffer
